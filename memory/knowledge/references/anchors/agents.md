## Mandatory Gates (CLAW Verbatim)

CLAW doctrine excerpt (integrated):

> 1. **Preflight gate**: environment, permissions, and dependencies checked.  
> 2. **Plan gate**: scope, constraints, and acceptance criteria captured.  
> 3. **Verification gate**: tests/lints/checks run and recorded.  
> 4. **Review gate**: regressions and risks explicitly assessed.  
>  
> No task may be marked `done` without all four gates.

## Lifecycle Contract (CLAW Verbatim)

CLAW doctrine excerpt (integrated):

> Every work item must move through these states:  
> `new -> clarified -> planned -> executing -> verifying -> review -> done | blocked | failed`

## Memory Rules

- Prefer links over duplication.
- Keep root anchors concise.
- Write durable facts to semantic or procedural layers, not working memory.
- Append dated outcomes to episodic memory.
- Promote repeated insights to semantic and repeated methods to procedural.

## Mem0 Non-Deletion Policy

Deleting memory from Mem0 is prohibited.
