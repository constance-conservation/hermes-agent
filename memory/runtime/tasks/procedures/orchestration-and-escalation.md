# Orchestration and Escalation

## Purpose

Unified procedure for orchestration flow, delegated execution, and escalation handling.

## CLAW Verbatim Principles

CLAW doctrine excerpt (integrated):

> 1. **Control plane first**  
>    Prefer one authoritative text interface for human intent.  
>
> 2. **State machine over terminal prose**  
>    Use explicit lifecycle states and typed events.  
>
> 3. **Planning and execution must be separated**  
>    Do not collapse planning and implementation into one implicit loop.

## Operating Flow

- Architect plans.
- Executor executes and verifies.
- Reviewer validates risk and completion quality.
- Orchestrator resolves blockers, routes escalation, and preserves audit trail.
