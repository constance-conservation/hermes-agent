SUB-PROMPT — MARKDOWN PLAYBOOK GENERATOR

Create a markdown file called AGENTIC_COMPANY_TEMPLATE.md for this workspace.

The markdown file must include:

1. company mission and operator objectives
2. org chart and hierarchy
3. role definitions
4. department definitions
5. project structure
6. reporting rules
7. memory and archival rules
8. task state definitions
9. evidence requirements for completion
10. escalation rules
11. board-of-directors review process
12. future channel architecture across WhatsApp, Telegram, and Slack
13. client onboarding questions
14. instructions for customizing the template per client
15. agent lifecycle and org hygiene rules
16. guidance for staying lean and avoiding unnecessary agents
17. role creation criteria
18. department activation criteria
19. reporting cadence guidance
20. summary format standards

WRITE STYLE

- clear
- reusable
- auditable
- minimal
- structured
- suitable for future deployment to client companies

Do not write it as commentary. Write it as an operational document for execution.

When generating supplemental markdown from this prompt in a governed deployment, place company-wide scoped outputs under `policies/core/governance/generated/` (indexed in `policies/core/governance/generated/README.md`), role-scoped durable work under `policies/core/governance/generated/by_role/<role_slug>/`, or project-only output under `operations/projects/<slug>/generated/`, per `policies/core/governance/artifacts-and-archival-memory.md` and `policies/core/governance/generated/by_role/README.md`.
